## 1. 模块边界

地图生成器只负责产出**地形 + HQ 坐标 + 桥梁**，不负责单位放置。

```
输入：地图尺寸（width, height）
输出：合法的 terrain 矩阵 + friendly_hq + enemy_hq
```

## 2. 硬性约束（必须满足）

| # | 约束 | 说明 |
|:--:|------|------|
| C1 | 尺寸范围 | 10×10 ~ 40×40，矩形 |
| C2 | HQ 可通行 | 两个 HQ 必须在可通行格上 |
| C3 | HQ 最小距离 | `chebyshev(HQ_A, HQ_B) ≥ max(8, min(width, height) × 0.7)` |
| C4 | 连通性 | 从 HQ_A 出发必须能走通到 HQ_B（BFS/DFS 校验） |
| C5 | 河流不可通行 | 河格移动消耗 = -1，不能放 HQ |
| C6 | 桥梁连通两岸 | 桥两侧的格子必须可通行 |
| C7 | 山脉不可通行 | 山地移动消耗 = -1，不能放 HQ，不可通行 |
| C8 | 河流宽度 | 1~3 格宽，随机决定 |

## 3. 地形生成规则

### 3.1 生成方式：噪声生成

使用 **Perlin 噪声 / Simplex 噪声** 生成地形分布，而非逐格随机。噪声生成保证同类地形自然集群——森林成片、山脉成岭、平原连贯。不存在孤立的单格障碍物。

```
噪声值映射：
  低 → 平原
  中 → 森林
  高 → 山地
  河流独立于噪声，由模板生成后覆盖
```

### 3.2 地形类型

| 地形 | 说明 |
|------|------|
| 平原 (0) | 基础地形，噪声低值区 |
| 森林 (1) | 噪声中值区，成片林地 |
| 山地 (2) | 噪声高值区，**不可通行**，成岭分布 |
| 河流 (3) | 模板生成，**不可通行**，1~3 格宽 |
| HQ (4) | 不参与噪声，生成完地形后放置 |
| 桥梁 (5) | 依赖河流，保证连通最少桥数 |

## 4. 连通性保证

```
从 friendly_hq 出发 BFS/DFS 遍历所有可通行格
河流和山脉视为障碍物
检查 enemy_hq 是否在可达集合中

→ 通过：导出 JSON
→ 失败：丢弃整张图，重新生成

重试下限：100 次（34×25 噪声生成速度快，100 次秒级完成）
超下限仍失败 → 在阻塞点自动加桥或削掉孤立山格
```

## 5. 输出格式

与现有 `data/maps/map_01.json` 兼容：

```json
{
    "map_size": { "width": 34, "height": 25 },
    "terrain": [[...], ...],
    "friendly_hq": { "x": 3, "y": 22 },
    "enemy_hq":    { "x": 30, "y": 2 }
}
```

不包含 `friendly_units` / `enemy_units` 字段（单位由其他模块负责）。
