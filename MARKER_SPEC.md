# 玩家标记系统 — 设计规格

> **版本**: v0.1  
> **状态**: 设计阶段  
> **依赖**: 已有兵种素材 (assets/units/*.png), pygame freetype

---

## 1. 概述

玩家无法看到地图上单位的真实位置。只能通过**拖放标记**来推测和记录。

标记面板位于战报面板下方，提供两类可拖放对象：
- **数字标签 (1-10)**：标识几号单位
- **兵种图标 (10 种)**：标识兵种 + 阵营

两者自由组合使用。所有标记拖走一个再生一个，无限供应。

## 3. 数字标签 (1-10)

### 3.1 生成方式

```
pygame freetype 渲染数字 1-10
  → 手写体 .ttf 字体（优先），无合适字体用 pygame 默认
  → 灰铅笔色 (#777766)
  → 透明背景 PNG
  → 尺寸约 36×36px（与调色板槽匹配）
```

### 3.2 使用方式

```
拖一个 "3" 到地图上某格 → 该格被标记为"3号单位在此"
拖走 → 槽内自动生成一个新的 "3"
地图上可存在多个 "3" → 玩家最新拖的那个代表当前推测位置
```

### 3.3 数量

每个数字 1-10 各一个初始副本。拖走即再生。

## 4. 兵种图标 (10 种)

### 4.1 来源

已有素材 `assets/units/` 下 10 张 2048×2048 PNG：

| 友军 (蓝) | 敌军 (红) |
|------|------|
| Infantry_blue.png | Infantry_red.png |
| Cavalry_blue.png | Cavalry_red.png |
| Artillery_blue.png | Artillery_red.png |
| Scout_blue.png | Scout_red.png |
| HQ_blue.png | HQ_red.png |

### 4.2 处理

```
已有 2048×2048 → pygame 缩放到 ~36×36 → 显示在托盘中
拖走 → 地图格上显示缩放后的图标 → 代表"该格有步兵(蓝)=友军步兵在此"
```

### 4.3 使用方式

```
拖蓝步兵到地图 + 拖数字 3 到同一格 → "3号友军步兵在这"
拖红骑兵到地图 + 拖数字 5 → "5号敌军骑兵在这"
```

## 5. 交互规则

| 操作 | 行为 |
|------|------|
| 从面板拖出 | 创建标记副本，槽内再生 |
| 放到地图格上 | 标记吸附到格子 |
| 拖到地图空白处/面板外 | 标记消失（删除） |
| 右键点击地图上的标记 | 删除 |
| 拖地图上已有标记 | 移动位置 |

## 6. 代码结构

```
src/ui/
  marker.py          ← 现有 MarkerSystem，需改造
  marker_palette.py  ← 新建：标记面板（数字 + 兵种图标）
  
src/core/constants.py
  → 新增: MARKER_TRAY_WIDTH, MARKER_NUM_FONT, MARKER_NUM_COLOR
```

### 6.1 marker_palette.py

```
class MarkerPalette:
    def __init__(rect, existing_unit_icons, num_font)
    def build_tray()          → 生成所有标记槽
    def handle_event(event)   → 拖拽检测
    def regenerate(slot_id)   → 拖走后再生
    def draw(screen)          → 渲染面板
```

### 6.2 与 MarkerSystem 的关系

```
MarkerSystem (现有) → 管理地图上的标记实例 (Marker)
MarkerPalette (新建) → 管理面板中的拖放源

MarkerSystem.handle_event() 处理地图上的标记操作
MarkerPalette.handle_event() 处理面板上的拖拽源
```
